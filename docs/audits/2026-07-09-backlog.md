Labels: `story`, `bug`, `foundation` · Branch: `feature/S4-scheduler-overlap-catalog-uniqueness`

### Requirements

- [ ] **Unique index on Track (artist, name)** (`ent/schema/track.go:139-142` is non-unique) + handle constraint conflicts in `getOrCreateTrack`/`MatchListens` by re-querying; add cleanup/merge migration for existing duplicate rows (which currently make `.Only()` fail permanently).

### Acceptance Criteria
- [ ] WHEN a sync/enrichment tick fires while the previous run for that user is active THEN the new run is skipped and logged
